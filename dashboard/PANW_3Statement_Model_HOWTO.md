# PANW Three-Statement Excel Model

`PANW_3Statement_Model.xlsx` is generated by the pipeline.
It is not committed to this repository because it requires first running the
full SEC EDGAR ingest + DuckDB warehouse build, which downloads ~50 MB of data.

## Generating the model

```bash
# One-time setup (3–5 min first run: installs cmdstan ~200 MB)
make setup

# Generate PANW warehouse then Excel model (~2–3 min after setup)
make ingest TICKER=PANW
make warehouse TICKER=PANW
make dashboard TICKER=PANW
```

The output lands at `dashboard/PANW_3Statement_Model.xlsx`.

## What the workbook contains

| Sheet                  | Contents                                                     |
|------------------------|--------------------------------------------------------------|
| Cover                  | Methodology, disclosure, and data sourcing notes            |
| Scenarios              | Base / Bull / Bear parameter overrides                       |
| Assumptions            | Active-scenario inputs (dropdown: Base/Bull/Bear)            |
| Income_Statement       | 12 historical + 4 forecast quarters                         |
| Balance_Sheet          | Accounting identity verified; BalanceCheck row = $0         |
| Cash_Flow              | Indirect method; GAAP OCF residual < $5M                    |
| Debt_Schedule          | Quarterly amortization                                       |
| Summary                | KPI dashboard (revenue, margins, DSO, DIO)                  |
| Sources                | Provenance table: accession_no + filing_url per data point  |
| Revenue_Disaggregation | ASC 280 Product vs. Subscription & Support (PANW only)      |

## Key properties to verify in the model

- **BalanceCheck row** (Balance_Sheet): every cell should read `$0`
- **Sources sheet**: every row has a non-blank `accession_no` and `filing_url`
- **Revenue_Disaggregation tab**: present for PANW (`has_physical_inventory=TRUE`),
  absent for CRWD (`has_physical_inventory=FALSE`)

## Running automated checks

```bash
# Unit tests for BalanceCheck, Inventory flag, OCF residual (no DB required)
make test

# Full pipeline end-to-end verification
make demo TICKER=PANW
```
